# Digital Cerebellum — 数字小脑架构设计

> 一个受生物小脑启发的轻量级预测-校正系统，作为大语言模型（LLM）的必要补充，
> 将 AI Agent 从"被动回答者"进化为"主动行动者"。

---

## 一、核心理念

```
┌─────────────────────────────────────────────────────────────────┐
│                      生物智能的双引擎架构                         │
│                                                                 │
│   大脑皮层 (Cerebral Cortex)    ←──→    小脑 (Cerebellum)        │
│   ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄       ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄           │
│   • 深层递归网络                 • 浅层前馈网络                    │
│   • 非线性计算                   • 线性计算                       │
│   • 慢速、灵活、上下文依赖        • 快速、精确、自动化              │
│   • 规划、推理、语言              • 预测、校正、时序                 │
│   • 占脑神经元 ~50%              • 占脑神经元 ~50%                 │
│                                                                 │
│   ≈ LLM / 大语言模型             ≈ Digital Cerebellum / 数字小脑   │
│   （已存在）                      （缺失的一半）                    │
└─────────────────────────────────────────────────────────────────┘
```

**关键洞察**：当前所有 AI Agent 框架（OpenClaw 等）都在给"大脑"装手脚，
但没有人构建"小脑"——那个让大脑能真正高效运作的预测-校正-自动化引擎。

## 二、生物学机制 → 工程组件映射

```
  生物小脑电路                              数字小脑组件
  ════════════                              ════════════

  苔藓纤维 (Mossy Fiber)                    感知层
  ──────────────────                        ──────────
  携带感觉/运动信号进入小脑        →         事件监听器，接收外部世界的信号


  颗粒细胞层 (Granule Cell Layer)           模式分离器
  ──────────────────────────                ──────────
  将低维输入映射到高维稀疏空间      →         将原始事件编码为高维向量
  核函数机器，做模式分离                      用于区分相似但不同的情境


  浦肯野细胞 (Purkinje Cell)                预测引擎
  ──────────────────────                    ──────────
  线性计算算法                    →         轻量前馈模型
  接收颗粒细胞的并行纤维输入                  给定状态+意图 → 预测结果
  是小脑皮层唯一的输出                        毫秒级推理，不调用 LLM


  攀爬纤维 (Climbing Fiber)                 误差信号
  ──────────────────────                    ──────────
  从下橄榄核传来误差信号           →         比较预测结果与真实结果
  驱动浦肯野细胞突触可塑性                    生成误差向量，驱动在线学习


  小脑深部核团 (Deep Cerebellar Nuclei)     决策路由器
  ──────────────────────────────            ──────────
  整合浦肯野细胞输出               →         判断置信度：
  向丘脑/大脑皮层发送信号                       高 → 直接执行（快路径）
                                               低 → 上报 LLM（慢路径）


  皮层-小脑回路                              皮层接口
  (Cortico-Cerebellar Loop)                 ──────────
  ──────────────────────                    反馈解耦：为 LLM 提供预测反馈
  双向闭环：皮层→桥脑→小脑→丘脑→皮层  →      任务巩固：将 LLM 的重复模式
  反馈解耦 + 任务巩固转移                     蒸馏为小脑本地模型
```

## 六、微区架构 — 通用小脑变换 (UCT)

生物学中，小脑的所有区域使用相同的电路结构，但通过连接不同的大脑皮层区域
来处理不同领域。数字小脑采用相同的设计：**一套计算原语，多个领域微区**。

```
                    ┌─────────────────────────────────┐
                    │        统一计算原语               │
                    │   (Unified Compute Primitive)    │
                    │                                  │
                    │   模式分离 → 预测 → 误差校正      │
                    │   (对所有微区完全相同)             │
                    └──────────────┬──────────────────┘
                                   │
               ┌───────────┬───────┼───────┬───────────┐
               │           │       │       │           │
               ▼           ▼       ▼       ▼           ▼
          ┌─────────┐ ┌────────┐ ┌─────┐ ┌─────┐ ┌─────────┐
          │ 对话微区 │ │文件微区│ │日程 │ │代码 │ │  ....   │
          │         │ │        │ │微区 │ │微区 │ │ 可扩展  │
          ├─────────┤ ├────────┤ ├─────┤ ├─────┤ ├─────────┤
          │I/O 适配 │ │I/O 适配│ │I/O  │ │I/O  │ │I/O 适配 │
          │消息收发  │ │文件读写│ │日历  │ │编辑器│ │  自定义 │
          │意图分类  │ │变更检测│ │提醒  │ │LSP  │ │         │
          └─────────┘ └────────┘ └─────┘ └─────┘ └─────────┘

  每个微区：
  • 共享同一套 [模式分离 → 预测 → 误差校正] 计算流程
  • 拥有独立的 I/O 适配器（决定输入从哪来、输出往哪去）
  • 拥有独立的权重（各自的学习互不干扰）
  • 可独立上线/下线（即插即用）
  • 新领域 = 新微区 = 新 I/O 适配器 + 初始化权重 + 开始学习
```

## 七、任务巩固流水线

这是数字小脑最独特的机制——让 LLM 处理的任务逐步"毕业"到小脑本地执行。

```
  阶段 0: 陌生任务
  ──────────────────────────────────────────────────
  首次遇到 → confidence ≈ 0 → 100% 走 LLM
  小脑只观察和记录

       │ 记录 input_pattern → llm_output
       ▼

  阶段 1: 模式积累
  ──────────────────────────────────────────────────
  同类任务出现 3~5 次 → 小脑开始识别模式
  confidence 上升到 0.3~0.5 → 仍然走 LLM
  但小脑的预测作为上下文提供给 LLM（反馈解耦）
  → LLM 推理更快、Token 消耗更少

       │ 模式稳定，误差收敛
       ▼

  阶段 2: 影子执行 (Shadow Execution)
  ──────────────────────────────────────────────────
  confidence 0.5~0.85 → 小脑和 LLM 同时预测
  以 LLM 结果为准，但比较两者差异
  持续训练小脑模型

       │ 小脑预测与 LLM 一致率 > 95%
       ▼

  阶段 3: 毕业 (Graduation)
  ──────────────────────────────────────────────────
  confidence >= 0.85 → 走快路径
  小脑独立执行，不再调用 LLM
  持续监控误差，如果环境变化导致误差上升
  → 自动降级回慢路径重新学习

       │ 持续运行
       ▼

  阶段 4: 固化 (Consolidation)
  ──────────────────────────────────────────────────
  长期稳定的任务 → 进一步优化模型
  可编译为规则/状态机，连神经网络推理都不需要
  成为真正的"条件反射"
```

## 八、流体记忆系统 (Fluid Memory)

生物记忆不是数据库——它是活的。每次回忆都会改变记忆本身（再巩固），
不被访问的记忆自然衰减（遗忘曲线），具体事件逐渐抽象为规律（模式提取），
短期记忆选择性转移为长期记忆（睡眠巩固）。

数字小脑的记忆系统复刻这些机制：

```
  ┌──────────────────────────────────────────────────────────────┐
  │                    流体记忆系统 (Fluid Memory)                │
  │                                                              │
  │  ┌────────────────────────────────────────────────────────┐  │
  │  │  感觉缓冲 (Sensory Buffer)                              │  │
  │  │  ─────────────────────────                              │  │
  │  │  原始事件流，秒级衰减                                     │  │
  │  │  大部分被丢弃，只有触发注意力阈值的进入短期记忆            │  │
  │  │  生命周期: ~5 秒                                         │  │
  │  │  存储: 内存环形缓冲区                                    │  │
  │  └──────────────────────────┬─────────────────────────────┘  │
  │                             │ 注意力筛选:                    │
  │                             │ 异常事件 / 高误差事件 /        │
  │                             │ 匹配已知模式 → 进入短期记忆     │
  │                             ▼                                │
  │  ┌────────────────────────────────────────────────────────┐  │
  │  │  短期记忆 (Short-Term Memory)                           │  │
  │  │  ─── 对应海马体：快速写入，容量有限 ───                  │  │
  │  │                                                          │  │
  │  │  每条记忆携带:                                            │  │
  │  │  • content    内容（文本/向量）                           │  │
  │  │  • strength   记忆强度 (0~1)                             │  │
  │  │  • embedding  语义向量                                   │  │
  │  │  • access_count  访问次数                                │  │
  │  │  • created_at / last_accessed  时间戳                    │  │
  │  │                                                          │  │
  │  │  衰减: 快速指数衰减                                       │  │
  │  │    strength *= exp(-λ₁ · Δt),  λ₁ 大                    │  │
  │  │                                                          │  │
  │  │  再巩固: 每次被检索时                                     │  │
  │  │    strength 重新激活到 0.8                                │  │
  │  │    embedding 向当前查询方向微调 (lerp, α=0.05)           │  │
  │  │    → 回忆不是"读取"，而是"带着当前视角重写"              │  │
  │  │                                                          │  │
  │  │  容量: ~50 条活跃记忆                                     │  │
  │  └──────────────────────────┬─────────────────────────────┘  │
  │                             │ 巩固条件:                      │
  │                             │ access_count >= 3 OR           │
  │                             │ 关联高误差事件 OR              │
  │                             │ 语义重要性高                   │
  │                             ▼                                │
  │  ┌────────────────────────────────────────────────────────┐  │
  │  │  长期记忆 (Long-Term Memory)                            │  │
  │  │  ─── 对应大脑皮层：慢速写入，提取规律 ───                │  │
  │  │                                                          │  │
  │  │  衰减: 亚线性慢衰减                                       │  │
  │  │    strength *= exp(-λ₂ · Δt^0.8),  λ₂ 小                │  │
  │  │    经常被访问的记忆几乎不衰减                              │  │
  │  │                                                          │  │
  │  │  形态演化:                                                │  │
  │  │    具体事件随时间被抽象为模式:                             │  │
  │  │    "周一 9:00 用户查邮件" ×5 → "工作日早上查邮件"        │  │
  │  │    细节消退，规律留存                                      │  │
  │  │    → 这不是信息丢失，是泛化能力的来源                      │  │
  │  │                                                          │  │
  │  │  再巩固: 同短期记忆，但修改幅度更小 (α=0.02)             │  │
  │  │                                                          │  │
  │  │  存储: 向量数据库 + SQLite                                │  │
  │  └──────────────────────────┬─────────────────────────────┘  │
  │                             │ 蒸馏:                          │
  │                             │ 反复出现的模式                 │
  │                             │ → 编译进预测引擎权重            │
  │                             ▼                                │
  │  ┌────────────────────────────────────────────────────────┐  │
  │  │  程序性记忆 (Procedural Memory)                         │  │
  │  │  ─── 对应小脑权重 ───                                    │  │
  │  │                                                          │  │
  │  │  = 预测引擎各微区的权重                                   │  │
  │  │  不再以"记忆"形式存在，而是变成了"能力"                   │  │
  │  │  你不"记得"怎么骑自行车，你只是"会"                      │  │
  │  │                                                          │  │
  │  │  通过在线学习（误差驱动）持续优化                          │  │
  │  │  通过 EWC 正则化防止灾难性遗忘                            │  │
  │  │  这是唯一天然就是"流动"的记忆层                           │  │
  │  │                                                          │  │
  │  │  存储: 模型权重文件 (.pt)                                │  │
  │  └────────────────────────────────────────────────────────┘  │
  │                                                              │
  │                                                              │
  │  ┌────────────────────────────────────────────────────────┐  │
  │  │  睡眠周期 (Sleep Cycle)                                 │  │
  │  │  ─── 离线维护，定期执行 ───                              │  │
  │  │                                                          │  │
  │  │  触发时机: 低负载时段 / 每日固定时间 / 显式调用           │  │
  │  │                                                          │  │
  │  │  执行步骤:                                                │  │
  │  │  ┌─────────────────────────────────────────────────┐    │  │
  │  │  │ 1. 衰减扫描                                     │    │  │
  │  │  │    清理 strength < 0.05 的记忆                   │    │  │
  │  │  │                                                  │    │  │
  │  │  │ 2. 巩固转移                                     │    │  │
  │  │  │    短期→长期（满足巩固条件的）                    │    │  │
  │  │  │    注意: 不是全部转移，只有对泛化有帮助的         │    │  │
  │  │  │                                                  │    │  │
  │  │  │ 3. 模式抽象                                     │    │  │
  │  │  │    长期记忆中的相似事件 → 聚类 → 提取规律         │    │  │
  │  │  │    原始事件的 strength 降低，规律的 strength 升高  │    │  │
  │  │  │                                                  │    │  │
  │  │  │ 4. 蒸馏检查                                     │    │  │
  │  │  │    成熟的模式→尝试编译进预测引擎（任务巩固）      │    │  │
  │  │  │                                                  │    │  │
  │  │  │ 5. 冲突消解                                     │    │  │
  │  │  │    矛盾的记忆→保留更新/更强的那个                 │    │  │
  │  │  └─────────────────────────────────────────────────┘    │  │
  │  └────────────────────────────────────────────────────────┘  │
  └──────────────────────────────────────────────────────────────┘
```

**记忆的完整生命周期**:

```
  外部事件
    │
    ▼
  感觉缓冲 ──(大部分丢弃)
    │
    │ 注意力筛选
    ▼
  短期记忆 ──(快速衰减，大部分遗忘)
    │
    │ 巩固筛选（被多次访问 / 高重要性）
    ▼
  长期记忆 ──(慢衰减，逐渐抽象化)
    │
    │ 蒸馏（模式成熟后）
    ▼
  程序性记忆 ──(编译为预测引擎权重)
    │
    │ 进一步固化
    ▼
  规则/状态机 ──(连推理都不需要，纯条件反射)

  每一层都在衰减、每一次访问都在修改。
  没有任何一条记忆是"存进去就不动了"的。
```

## 十一、神经科学修正（v2 架构升级）

基于对 2024-2026 最新神经科学文献的审计，v1 架构存在多处过度简化。
以下修正按优先级分层，Phase 0 必须在接口设计中预留位置。

### 核心修正（Phase 0 必须设计接口）

```
  修正 A: 群体编码 — 多头预测替代单头预测
  ════════════════════════════════════════════════════════════

  v1:  预测引擎输出一个向量 + 一个 sigmoid 置信度
  v2:  K 个并行预测头（K=4~8），各自独立预测

  依据: 2025 J.Neuroscience — 浦肯野细胞群体的空间相关性动态
       （而非单细胞输出）才是小脑编码行为信息的方式

  效果:
  • 置信度从"自我报告"变为"涌现"
    一致 → 高置信度 / 分歧 → 低置信度
  • 天然的不确定性量化，不需要额外模型
  • 可区分"确定的部分"和"不确定的部分"

  实现: K 个独立的线性头，共享模式分离器输出


  修正 B: 三种误差信号
  ════════════════════════════════════════════════════════════

  v1:  只有感觉预测误差（预测结果 vs 实际结果）
  v2:  三种并行误差通道

  依据: 2025 Nature Communications — 小脑攀爬纤维携带奖励预测误差
       2025 J.Neuroscience — 小脑深部核团编码时序预测误差

  三种误差:
  ┌──────────────────────────────────────────────────────────┐
  │  sensory_error   预测结果 vs 实际结果   → 更新预测引擎    │
  │  temporal_error  预测时间 vs 实际时间   → 更新节律系统    │
  │  reward_error    预期效果 vs 用户反馈   → 更新路由器偏好  │
  └──────────────────────────────────────────────────────────┘

  效果: 系统不只学"做什么"，还学"什么时候做"和"做得好不好"


  修正 C: 明确计算边界
  ════════════════════════════════════════════════════════════

  依据: 2025 ArXiv — 小脑的三个硬约束:
       预测性 / 短时间尺度 / 连续性

  数字小脑 NOT 做什么:
  ┌──────────────────────────────────────────────────────────┐
  │  ✗ 长期规划（> 5秒的决策 → 交给 LLM）                    │
  │  ✗ 离散分类推理（"这是什么" → 分类器或 LLM）             │
  │  ✗ 开放式文本生成（写文章 → LLM）                        │
  │  ✗ 多步逻辑推理（证明/推导 → LLM）                       │
  │                                                          │
  │  ✓ 短时间窗口预测（100ms ~ 5s）                          │
  │  ✓ 连续信号的模式匹配和预测                               │
  │  ✓ 基于历史的快速决策                                     │
  │  ✓ 误差驱动的在线学习                                     │
  └──────────────────────────────────────────────────────────┘
```

### 重要修正（Phase 1 实现）

```
  修正 D: 多位点学习
  ════════════════════════════════════════════════════════════

  v1:  只有预测引擎权重被更新
  v2:  4 个可塑性位点独立学习

  依据: 2024 Nature Communications / eLife — 小脑至少 4 个可塑性位点

  位点 1: 预测引擎权重                         ← v1 已有
  位点 2: 路由器权重（学习何时信任小脑）         ← 新增
  位点 3: 频率滤波器参数（学习信号带宽）         ← 新增
  位点 4: 模式分离器反馈增益（学习稀疏度）       ← 新增


  修正 E: 深部核团 → 主动计算层
  ════════════════════════════════════════════════════════════

  v1:  决策路由器 = if confidence > threshold
  v2:  有自身可塑性的计算模块

  依据: 2025 Frontiers — 深部核团做速率编码转换、
       输出同步、延迟补偿、拥有独立可塑性

  功能:
  • 将多头预测的不规则输出转换为平滑决策
  • 多微区输出的同步与整合
  • 延迟补偿（根据执行层延迟提前发出指令）
  • 自身权重通过 reward_error 更新
```

### 优化修正（Phase 2 实现）

  修正 H: 状态估计器
  ════════════════════════════════════════════════════════════

  依据: 2026 Nature Communications — 小脑某些区域编码
       ground-truth 自身运动，不只是预测

  加入与预测引擎并行的状态估计模块:
  不做预测，做"此刻到底发生了什么"的准确感知

## 十三、应用场景与路线图

### 场景 A: LLM 操作预评估（首要场景）

```
  问题: AI Agent 幻觉导致企业年损失数百亿美元
       62% 企业在实验 Agent，但只有 14% 投入生产
       工具调用错误: 参数错误 68%、选错工具 21%、编造数据 11%

  现有方案: 用另一个 LLM 检查 → 贵 (又一次推理)、慢 (1-3s)、也可能幻觉
  我们的方案: 轻量小脑预测操作后果 → 几乎零成本、<10ms、从历史中学习

  工作方式:
  ┌─────────────────────────────────────────────────────────┐
  │  LLM Agent 准备调用工具:                                 │
  │  tool_call("send_email", {to: "张三", body: "..."})     │
  │                      │                                   │
  │                      ▼                                   │
  │  ┌─ 数字小脑 (LLM预评估微区) ───────────────────────┐   │
  │  │  输入: tool_name + params + 对话上下文              │   │
  │  │  预测: { success_prob: 0.92, risk_type: "none" }   │   │

## 十五、科学基础审计

本项目的每一个核心映射都需要文献支撑。以下逐条标注科学可信度，
诚实区分"有直接文献"、"合理简化"和"需要注意"的部分。

### 有直接文献支撑的映射

```
  映射 1: 颗粒细胞层 → 核函数机器 / Random Fourier Features
  ════════════════════════════════════════════════════════════

  可信度: ★★★★★ 最强映射

  文献:
  • Bhalla (2022) Frontiers in Computational Neuroscience
    "Cerebellum as a kernel machine"
    直接论证颗粒细胞做高维扩展编码 (expansion recoding)，
    使浦肯野细胞的线性读出在高维空间中足够强大。

  • Bhatt et al. (2025) Scientific Reports
    "A computational model of the cerebellar granular layer
     calibrated to experimental data"
    生物校准模型验证了 Golgi 细胞抑制反馈如何优化模式分离性能，
    中等抑制水平最优——直接支持我们的 Golgi 门控设计。

  我们的实现:
    RFF (Random Fourier Features) 近似 RBF 核函数
    = 颗粒细胞的随机连接 + 稀疏激活的工程对应

  是否有简化: 是。
    生物颗粒细胞的扩展是否完全随机仍有争议。
    文献指出可能存在结构化编码成分，但随机扩展作为
    第一近似是合理的，且 RFF 已被大量工程实践验证。


  映射 2: 攀爬纤维 → 多种误差信号
  ════════════════════════════════════════════════════════════

  可信度: ★★★★★

  文献:
  • Kostadinov & Bhatt (2025) Nature Communications
    攀爬纤维携带奖励预测误差 (RPE)，不只是感觉预测误差。

  • 2025 J.Neuroscience
    深部核团编码时序预测误差 (TPE)。

  • Sendhilnathan et al. (2025) J.Neuroscience
    小脑在人类强化学习中编码预测误差。

  我们的实现:
    三通道误差 (SPE / TPE / RPE) 分别驱动不同组件的学习。

  是否有简化: 轻微。
    生物中误差信号的种类可能更多，且传导通路更复杂。
    三种是基于当前文献的合理归纳。


  映射 3: 小脑→皮层任务巩固 → 知识蒸馏
  ════════════════════════════════════════════════════════════

  可信度: ★★★★★

  文献:
  • Boven et al. (2024.12) Nature Communications
    "Cerebellar-driven cortical dynamics can enable task
     acquisition, switching and consolidation"
    直接描述: 小脑快速学会新任务 → 逐步把知识转移给皮层 →
    皮层巩固后小脑释放容量。
    与我们的"任务巩固流水线"几乎同一个思路。

  我们的实现:
    小脑(预测引擎)从 LLM 输出中学习 → 成熟后蒸馏为本地模型 →
    该领域"毕业"到快路径。

  是否有简化: 有。
    生物中巩固涉及睡眠期的离线重放，时间尺度是天到周。
    我们的巩固可以更快（小时级），因为数字系统无此限制。


  映射 4: 浦肯野细胞群体 → 多头预测 + 置信度涌现
  ════════════════════════════════════════════════════════════

  可信度: ★★★★☆

  文献:
  • 2025 J.Neuroscience
    浦肯野细胞群体的空间相关性动态（而非单细胞输出）
    编码行为信息。

  • Cirtala et al. (2025)
    "Contextual effects during sensorimotor adaptation are an
     emergent property of population coding in a cerebellar-
     inspired model"
    验证群体编码在小脑启发模型中的功能。

  我们的实现:
    K 个独立线性头 → 一致性涌现为置信度。

  是否有简化: 有。
    生物浦肯野细胞群体有 ~10-20 个细胞协同编码，且它们之间
    有通过中间神经元的侧抑制。我们的 K 个头是完全独立的，
    缺少侧抑制。Phase 2 可考虑加入头间竞争机制。


  映射 5: 小脑参与认知/非运动功能
  ════════════════════════════════════════════════════════════

  可信度: ★★★★☆

  文献:
  • Clausi et al. (2024) Springer — 系统综述
    "The Role of the Human Cerebellum for Learning from and

    确认小脑在非运动反馈学习中有重要作用。
    但约 50% 的小脑患者在非运动学习上仍然正常，

  • 2025 Nature Communications
    小脑在人类强化学习中编码预测误差（认知任务）。

    小脑不只做运动控制，也做认知预测。

### 合理但需标注为简化的映射

```
  映射 6: LLM ≈ 大脑皮层
  ════════════════════════════════════════════════════════════

  可信度: ★★★☆☆ — 功能类比，非结构类比

  支持的证据:
  • Ren et al. (2025) ArXiv
    "A Brain-like Synergistic Core in LLMs"
    LLM 自发产生类似大脑的"协同核心"——中间层做信息整合，
    两端做冗余处理。这种组织结构在随机初始化中不存在，
    通过学习涌现。

  • Whittington et al. (2025)
    新皮层和小脑都用预测误差学习构建内部世界模型，
    与注意力机制有功能交集。

  但 LLM 不是大脑皮层:
  • LLM 是 Transformer（并行注意力），皮层是递归网络
  • LLM 无在线学习，皮层有突触可塑性
  • LLM 无工作记忆（靠 context window 模拟），皮层有真正的工作记忆
  • LLM 处理离散 token，皮层处理连续信号

  结论:
    功能层面的类比（慢、贵、灵活 vs 快、轻、自动化）成立。
    不应声称 "LLM 就是大脑皮层"，而应说
    "LLM 在 AI Agent 系统中扮演大脑皮层的功能角色"。


  映射 7: 浦肯野细胞 → 线性计算
  ════════════════════════════════════════════════════════════

  可信度: ★★★☆☆ — 经典结论，但正在被修正

  支持的证据:
  • Walter & Khodakhah (2006/2009) PNAS
    经典实验验证浦肯野细胞的输入-输出函数为线性。

  修正的证据:
  • Cirtala & De Schutter (2024) bioRxiv
    发现浦肯野细胞树突有分支特异的非线性（P型钙通道
    引发的钙尖峰），挑战纯线性假说。

  • Hao et al. (2024) PNAS
    "A vector calculus for neural computation in the cerebellum"
    提出更复杂的向量计算框架。

  对我们的影响:
    Phase 0 的纯线性模型（nn.Linear）是过度简化但可用的起点。
    Phase 1 的浅层网络（Linear → ReLU → Linear）
    恰好能捕捉有限的树突非线性。
    我们的分阶段策略意外地与最新文献吻合。


  映射 8: EWC → 防灾难性遗忘
  ════════════════════════════════════════════════════════════

  可信度: ★★★☆☆ — 工程有效，非生物忠实

  生物学中的真实机制:
  • Boven et al. (2024) Nature Communications
    小脑防止灾难性遗忘的方式不是权重正则化，
    而是通过皮层-小脑回路的系统巩固:
    小脑快速学 → 知识转移给皮层 → 释放自身容量。
    这是一种结构性解决方案，不是参数层面的正则。

  EWC 的定位:
  • 功能上解决的是同一个问题（保持旧知识 + 学习新知识）
  • 作为 Phase 0 的工程近似是合理的
  • 但不应声称 "EWC 是生物启发的防遗忘机制"
  • 更忠实的做法是我们的任务巩固流水线 + 流体记忆睡眠周期
    （Phase 1 实现），这才真正对应系统巩固
```

### 需要注意的边界条件

```
  约束: Tsay & Ivry 2025 的三个硬约束
  ════════════════════════════════════════════════════════════

  重要性: ★★★★★ — 定义了小脑架构的适用边界

  文献:
  • Tsay & Ivry (2025) ArXiv 2509.09818
    "Cerebellar Contributions to Action and Cognition:
     Prediction, Timescale, and Continuity"
    提出三个约束条件:

    1. 预测性 — 小脑做预期性计算，不是反应性计算
    2. 短时间尺度 — 预测限于短时间窗口
    3. 连续性 — 小脑处理连续表征（空间、时间）

  • McDougle, Tsay, Taylor, Ivry (2020)
    实验证据: 小脑萎缩患者在连续任务（心理旋转）上受损，
    但在离散任务（符号搜索）上正常。

  对我们的影响:

    约束 1 (预测性): ✅ 完全满足
    我们的核心就是预测（预测 tool_call 结果、预测用户意图）

    约束 2 (短时间尺度): ✅ 完全满足
    我们明确限制: 预测窗口 100ms ~ 5s，长期规划交给 LLM

    约束 3 (连续性): ⚠️ 需要确保设计正确
    tool_call 预评估表面上是离散决策（安全/不安全）。
    但我们的计算流程底层是连续的:

    tool_call → 嵌入空间(连续) → RFF映射(连续)
    → 多头预测(连续向量) → 群体一致性(连续)
    → 路由决策(离散: 最终才二值化)

    关键设计原则:
    ┌──────────────────────────────────────────────────┐
    │  预测引擎必须输出连续的结果向量                     │
    │  （"预测这个操作会产生什么后果"）                   │
    │  而不是直接输出分类标签                             │
    │  （"这个操作安全/不安全"）                          │
    │                                                    │
    │  安全/不安全是从连续置信度中派生的                   │
    │  不是模型直接输出的                                 │
    │                                                    │
    │  如果做成分类器 → 违反连续性约束 → 类比不成立       │
    │  如果做成结果预测器 → 满足连续性 → 类比成立         │
    └──────────────────────────────────────────────────┘

    MOBA 场景: ✅ 完美满足全部三个约束
    （位置、时序、力度全是连续的）
```

## 十七、诚实边界与风险

### 我们不应该声称的

```
  ✗ "LLM 就是大脑"
    → 应说: "LLM 在 Agent 系统中扮演大脑皮层的功能角色"

  ✗ "EWC 是生物启发的防遗忘机制"
    → 应说: "EWC 是工程近似，生物用系统巩固（我们的任务巩固
             流水线才是更忠实的对应）"

  ✗ "只有我们想到了这个"
    → 应说: "每个组件都有人做过，我们的创新在于
             用小脑第一性原理组合它们并应用于 LLM Agent"

  ✗ "我们的架构完全忠实于生物小脑"
    → 应说: "我们选择性地映射了经过验证的计算原理，
             同时在工程可行性上做了明确的简化"
```

### 竞争风险

```
  风险 1: 方向正在收敛
  ──────────────────────────────────────────────────
  2026.02 ArXiv 论文已经把小脑原理用到 RL Agent 在线纠错。
  时间窗口不会永远开着。
  → 应对: 快速完成 Phase 0，用可验证的数据建立先发优势。

  风险 2: 竞争对手不需要小脑也能解决问题
  ──────────────────────────────────────────────────
  ToolSafe 用多任务 RL 就把有害 tool_call 减少了 65%。
  RouteLLM 用路由器就把成本降了 2x。
  我们的优势不是"能不能做到"，而是:
  • 做得更好（<10ms vs 1-3s、在线学习 vs 静态）
  • 更强的理论解释力（可发表的学术贡献）
  • 更大的扩展性（同一架构从 tool_call 到 MOBA）

  风险 3: 生物忠实度是双刃剑
  ──────────────────────────────────────────────────
  如果某个纯工程 trick 效果更好但不符合生物学，怎么办？
  → 原则: Phase 0 验证核心假设。
    如果生物启发的设计在关键指标上（准确率、延迟、学习速度）
    优于纯工程方案 → 继续沿着生物学走。
    如果不优于 → 诚实地说"类比到此为止"，
    保留有效的工程组件，放弃不如工程方案的生物映射。
```

### Phase 0 必须回答的核心问题

```
  Q1: 群体编码的置信度是否比 sigmoid 更可靠？
      验证方式: confidence 与实际准确率的相关系数
      基线: 单头 sigmoid 置信度

  Q2: 小脑启发的在线学习是否比 ToolSafe 的 RL 更好？
      验证方式: 相同数据量下的安全检测准确率
      基线: ToolSafe 报告的 65% 减少率

  Q3: <10ms 延迟是否在真实 tool_call 场景中成立？
      验证方式: 端到端延迟测量（含嵌入计算）
      基线: ToolSafe / LlamaFirewall 的延迟

  Q4: 在线学习是否真的让系统越用越准？
      验证方式: 学习曲线（准确率 vs 样本数）
      基线: 静态模型（训练后冻结）

  如果 Q1-Q4 全部为"是" → 核心假设验证通过，继续 Phase 1
  如果任何一个为"否" → 分析原因，调整设计或诚实缩小 claim
```

## Phase 4-6 架构扩展（2026-03-04 补充）

> 以下内容记录 Phase 4（技能习得）、Phase 5（通用微区）、Phase 6（微操引擎）的架构设计。
> 所有 Phase 均已实现并通过测试（225 tests passing）。

### Phase 4：技能习得与自主执行

**核心洞察**：小脑不只是"判断安全不安全"，它要能**替代 LLM 直接执行**已学会的技能。

```
生物学映射：
  程序性记忆（小脑 → 深部核团）
  → 反复练习某个动作后，小脑能直接执行，不再需要皮层参与
  → 骑自行车、打字、弹钢琴——你不需要"想"怎么做

数字实现：
  SkillStore（程序性记忆）
  → 每次 LLM 回答后，小脑将 (输入模式 → 响应/动作) 存为技能
  → 下次遇到相似输入，直接从技能库检索并执行
  → 成功的技能被强化，失败的技能被消退
  → 睡眠周期合并相似技能、淘汰弱技能
```

**架构**：

```
用户输入
    │
    ▼
  SkillStore.match(embedding)
    │
    ├─ 命中 (similarity > 0.85, confidence > 0.4)
    │    → 直接执行技能（文本回复 或 工具调用序列重放）
    │    → 不调 LLM，延迟 < 10ms
    │
    └─ 未命中
         → 调 LLM（慢路径）
         → 将 LLM 的响应学习为新技能
         → skill_feedback(success/fail) 驱动强化/消退
```

**关键参数**：
- `SIMILARITY_THRESHOLD = 0.85` — 匹配阈值
- `MIN_CONFIDENCE = 0.4` — 最低执行置信度
- `REINFORCEMENT_RATE = 0.1` — 成功后置信度增长率
- `EXTINCTION_RATE = 0.15` — 失败后置信度衰减率

**验证结果**：20 次交互后 25% 自动化率（Demo 验证）。

### Phase 5：通用微区框架

**核心洞察**：生物小脑的微区是**基因决定**的（不是经验涌现），但微区内部的突触权重通过经验特化。

```
生物学：
  小脑有 ~5000 个微区，每个处理特定的感觉-运动回路
  微区的存在由基因编程（跨个体一致、跨物种保守）
  微区的内部连接通过经验学习

数字实现：
  预定义 6 个通用微区（覆盖 Agent 主要操作领域）
  每个微区共享同一个小脑引擎（RFF → K-head → Router）
  通过经验特化各自的预测精度
```

**6 个预定义微区**：

| 微区 | 域 | TaskHead | 快路径检测 |
|------|-----|----------|-----------|
| ToolCallMicrozone | tool_call | safety | — |
| PaymentMicrozone | payment | payment_risk | 金额 > $10k |
| ShellCommandMicrozone | shell_command | shell_safety, shell_destructive | rm -rf, sudo, chmod 777 等 |
| FileOperationMicrozone | file_operation | file_safety, file_sensitivity | /etc/passwd, .env 等敏感路径 |
| APICallMicrozone | api_call | api_safety, api_data_leak_risk | DELETE 方法, Authorization 头 |
| ResponsePredictionMicrozone | response_prediction | response_predictability, response_complexity | 查询类型判断 |

### Phase 6：微操引擎（Micro-Operation Engine）

**核心洞察**：生物小脑的**本职工作**是连续运动控制（200Hz），不是认知判断。
Phase 0-5 实现了小脑的认知功能，Phase 6 实现了小脑的运动控制功能。

```
生物学映射：

  运动皮层 → "我要拿杯子"（意图）
       │
  小脑接收传出副本 → 前向模型预测"手会移动到哪里"
       │
  感觉反馈 → 实际位置
       │
  SPE = 预测 - 实际 → 误差信号
       │
  攀爬纤维 → 在线调整前向模型
       │
  循环 (200Hz): observe → predict → act → compare → learn
```

**架构**：

```
环境 (60Hz+)
    │ state (numeric vector)
    ▼
┌─ MicroOpEngine ───────────────────────────────────────┐
│                                                         │
│  StateEncoder                                           │
│    数值状态 → 归一化向量                                   │
│    Welford 在线均值/方差（无需完整数据集）                   │
│    直接模式 (<0.1ms) 或 投影模式 (<0.5ms)                  │
│       │                                                 │
│  PatternSeparator (复用 Phase 0 的 RFF 层)               │
│    低维状态 → 高维稀疏编码                                 │
│       │                                                 │
│  ActionNet (浅层 MLP + Tanh)                             │
│    稀疏编码 → 动作向量 ∈ [-1, 1]^action_dim               │
│       │                                                 │
│  ForwardModel (残差 MLP)                                 │
│    (state, action) → Δstate                             │
│    predicted_next = state + Δstate                       │
│       │                                                 │
│  ActionEncoder                                          │
│    归一化 ↔ 原始动作空间                                   │
│       │                                                 │
│  在线学习                                                │
│    ForwardModel.learn(state, action, actual_next)        │
│    ActionNet 策略梯度 (reward × SPE modulation)           │
│                                                         │
└─────────────────────────────────────────────────────────┘
    │ action
    ▼
环境.execute(action) → reward
```

**关键设计决策**：

1. **残差预测**：ForwardModel 预测 Δstate 而非绝对 next_state，学习更容易
2. **SPE 调制学习率**：预测误差大时学得更快（匹配攀爬纤维的 burst firing）
3. **浅层前馈网络**：不用 RNN/Transformer，匹配生物小脑的前馈架构
4. **梯度裁剪**：`clip_grad_norm_(1.0)` 防止不稳定

**验证结果**：

| 指标 | 目标 | 实际 |
|------|------|------|
| 运行频率 | 60Hz | 285Hz |
| 单步延迟 | <16ms | 3.5ms |
| 前向模型学习 | SPE 下降 | SPE ↓99% |
| LLM 调用 | 零 | 零 |

### 统一架构：DigitalBrain 双模式

Phase 6 完成后，`DigitalBrain` 支持两种操作模式：

```
DigitalBrain
    │
    ├─ brain.think(text)     → 文本模式（Phase 0-5）
    │    SkillStore → match? → 执行/LLM
    │    输入: 自然语言
    │    输出: 文本 + 工具调用
    │    延迟: 10ms (fast) / 1-5s (slow)
    │
    └─ brain.control(env)    → 控制模式（Phase 6）
         MicroOpEngine → observe → predict → act → learn
         输入: 数值状态向量 (60Hz+)
         输出: 连续动作向量
         延迟: 3.5ms/step
```

两种模式共享相同的小脑原理（RFF 模式分离、误差驱动学习、在线适应），
但在不同时间尺度和输入模态上运行。

**这就是"给 LLM 一个身体"的完整含义：**
- `think()` = 小脑的认知功能（预测、判断、技能执行）
- `control()` = 小脑的运动功能（连续控制、精密操作、实时学习）
