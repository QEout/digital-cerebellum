## 一、我们到底在做什么

### 错误的叙事（已放弃）

~~"给 AI Agent 加个安全检查器"~~
- 这把小脑矮化成了一个二元分类器
- 与 AgentGuard、Fiddler、RADIUS 等竞品没有本质区别
- "更快更便宜"的价值主张随着 LLM 降价和 prompt cache 技术会自然贬值

### 正确的叙事（当前方向）

> **LLM 是瓶子里的大脑——能思考，但不能行动。**
> **小脑给它一个身体——让它能积累经验、形成反射、自主成长。**
> **这不是"更快的 LLM"，这是一种全新的 AI 架构。**

核心价值不是"更安全"或"更便宜"，而是：

**让 AI Agent 从"回答机器"变成"有经验的行动者"。**

## 二、LLM 的本质局限（prompt cache 也解决不了的）

| 局限 | 说明 | 小脑如何解决 |
|------|------|-------------|
| **无状态** | 每次对话从零开始，用了一年跟第一天一样 | 小脑是持久状态，跨会话积累经验 |
| **纯反应式** | 永远等输入才动，不会预判和主动行动 | 小脑预测下一步需求，提前准备 |
| **无身体** | 每个动作都要"想一步说一步"，没有肌肉记忆 | 小脑学会动作序列，整套自动执行 |
| **无成长** | 模型冻结，对所有用户相同 | 小脑为每个场景积累独特经验 |

prompt cache 只能缓存 token，不能缓存"经验"。
模型降价只影响单价，不改变"每次都要完整推理"的本质。

## 三、三个核心能力目标

### 3.1 低延迟高效操作 ✅ 已实现

- 快路径 <10ms，benchmark 验证
- RFF 模式分离 + K-head 线性预测 = 数学上保证快

### 3.2 自主学习 ✅ 已实现（Phase 4）

- 在线学习（从 LLM 反馈中学习预测模型）
- 三通道误差信号（SPE/TPE/RPE）
- 记忆巩固（睡眠周期）
- EWC 防遗忘
- **SkillStore 程序性记忆** — 存储 (输入模式 → 输出动作/响应) 对
- **技能匹配** — 向量相似度检索，找到最匹配的已学技能
- **技能执行** — 高置信度时直接执行，不调 LLM
- **强化/消退** — 成功的技能增强，失败的技能减弱
- **睡眠巩固** — 合并相似技能，淘汰不可靠技能
- Demo 验证：20 次交互后 25% 自动化率

### 3.3 自由探索 ✅ 已实现（Phase 4）

- 好奇心驱动（追踪学习进展，识别高学习潜力领域）
- 探索排名（`get_curiosity_ranking()`）
- **主动探索信号** — `get_exploration_requests()` 输出可执行的探索建议
- **DigitalBrain.get_exploration_suggestions()** — 大脑级别的探索驱动

### 3.4 精密微操 ✅ 已实现（Phase 6）

**这是生物小脑最核心的能力。** 小脑的"本职工作"是精密运动控制——
认知功能（预测、安全检查）反而是近年才被广泛认识的"副业"。

**已实现：**
- **MicroOpEngine** — 连续控制循环：observe → predict → act → compare → learn
- **StateEncoder** — 数值状态向量直接进入 RFF（绕过 sentence-transformer，<0.1ms）
- **ForwardModel** — 前向模型 (state + action) → predicted_next_state（浅层前馈 MLP，匹配生物学）
- **ActionEncoder** — 连续动作空间编码/解码（归一化、pad/truncate）
- **PatternSeparator 复用** — 同一个 RFF 稀疏编码层，用于连续状态空间
- **在线学习** — 从感觉预测误差 (SPE) 实时更新前向模型
- **策略网络** — 基于奖励信号的在线策略梯度学习
- **两个验证环境** — TargetTracker（2D 追踪）、BalanceBeam（平衡杆）
- **Demo 验证**：285 Hz 实际运行速度，3.5ms/step，SPE 降低 99%

**应用场景：**

```
┌─────────────────────────────────────────────────────────┐
│  场景          大脑（LLM）说        小脑（微操）做        │
│  ─────────    ──────────────       ──────────────       │
│  MOBA         "团战集火ADC"        闪现+技能连招+走A      │
│  FPS          "守住B点"            压枪+预瞄+身法        │
│  Figma        "做一个登录页"        8px网格对齐+间距一致   │
│  代码          "重构这个模块"        精确的AST变换+重命名   │
│  机器人        "把杯子放桌上"        手臂轨迹+力度+平衡     │
│  钢琴          "弹月光奏鸣曲"        每个手指的力度和时机    │
└─────────────────────────────────────────────────────────┘
```

**技术需求：**
- 连续推理模式（streaming input/output，非 request-response）
- 动作空间编码（把连续动作映射为向量）
- 实时前向模型（预测动作的感觉后果）
- 序列学习（学习多步动作序列作为一个"技能单元"）
- 时序精度（毫秒级动作协调）

**生物学映射：**

| 生物学 | 数字系统 |
|--------|---------|
| 运动皮层发出运动指令 | LLM 发出高层意图 |
| 小脑接收传出副本 | 小脑接收 LLM 的意图 |
| 小脑通过前向模型预测感觉后果 | 小脑预测动作结果 |
| 小脑输出精细运动修正 | 小脑输出具体操作序列 |
| 攀爬纤维传递误差 | 执行结果反馈驱动学习 |
| 反复练习 → 程序性记忆 | 在线学习 → 自动化执行 |

**这是我们与"小 LLM 做 System 1"方案最根本的区别：**
小 LLM 能做"快速判断"，但做不了"精密控制"。
只有小脑架构——浅层前馈、实时闭环、序列学习——才能做到微操。

### 已纠正的错误

**错误：小脑微区可以通过经验自主涌现**

> 事实：小脑微区是基因决定的，不是经验涌现的。
> "The molecular identity of individual Purkinje cells seems to be
> set and independent of cerebellar connectivity."
> — Nature Reviews Neuroscience

- 微区结构在出生前由基因编程，在个体间高度一致、跨物种保守
- 经验只改变微区**内部**的突触权重，不改变微区的**存在与否**
- 正确做法：预定义通用微区 → 通过经验特化（而非从零涌现）

### 符合生物学的技能习得模型

来自 2025 Nature Communications 的运动学习三阶段：

```
阶段 1（早期）: 小脑主导
  → 快速检测误差，调整动作
  → 对应：慢路径，LLM 处理，小脑学习

阶段 2（中期）: 皮层-小脑协同
  → 双向交互，逐步内化
  → 对应：shadow 路径，小脑预测 + LLM 验证

阶段 3（晚期）: 技能自动化
  → 小脑抑制减少，皮层自主执行
  → 对应：快路径接管，不需要 LLM

记忆巩固: 从小脑皮层 → 小脑核团
  → 从复杂灵活系统 → 简单快速系统
  → 对应：睡眠周期将技能压缩为程序性记忆
```

## 六、Agent 瓶颈分析

2026 年生产环境中 90% 的 Agent 失败。主要瓶颈：

| 瓶颈 | 严重度 | 小脑能解决 | 机制 |
|------|--------|-----------|------|
| **死循环** | 致命 | ✅ | 检测重复失败模式，立即终止并上报 |
| **上下文污染** | 严重 | ✅ | 小脑记忆在 context window 外，零 token 开销 |
| **参数幻觉** | 严重 | ✅ | 学习有效参数模式，预测合法性 |
| **无状态** | 严重 | ✅ | 小脑就是跨会话持久状态 |
| **成本** | 中等 | ✅ | 快路径绕过 LLM |
| **规划错误** | 中等 | ⚠️ | 可预测单步风险，无法评估计划逻辑一致性 |

## 七、竞争定位

### 与静态护栏对比（AgentGuard, Fiddler, RADIUS, MCPShield）

- 它们：静态规则，不学习
- 我们：在线学习，越用越准
- 它们：规则由人编写
- 我们：从 LLM 行为中自动学习

### 与双系统 AI 对比（CogRouter, SOFAI, Talker-Reasoner）

- 它们用小 LLM 做 System 1：100ms+，仍然昂贵
- 我们用小脑电路做 System 1：5ms，在线学习
- 它们需要定制训练数据
- 我们从任何 LLM 的输出中在线学习

### 与 prompt cache 对比

- prompt cache：缓存 token（记忆）
- 我们：学习模式（经验）
- prompt cache：完全相同的查询才命中
- 我们：相似模式即可泛化

## 八、下一步实施计划

### Phase 4：技能习得与执行 ✅ 已完成

**目标：让快路径不只是"判断"，而是"执行"。**

已实现：

1. **SkillStore** (`memory/skill_store.py`) — 程序性记忆，存储 (输入 → 响应/动作) 对
2. **技能匹配** — 向量余弦相似度检索，找到最匹配的已学技能
3. **DigitalBrain.think() 重写** — 先查技能库，命中则直接执行，否则调 LLM 并学习
4. **动作序列重放** — 多步工具调用打包为技能单元，整套重放
5. **强化/消退** — `skill_feedback()` 驱动技能置信度变化
6. **主动探索** — `get_exploration_requests()` 输出可执行探索信号
7. **睡眠巩固** — 合并相似技能，淘汰弱技能
8. **26 个测试全部通过**，Demo 验证 25% 自动化率

### Phase 5：通用微区框架 ✅ 已完成

**目标：预定义一组覆盖面广的通用微区（符合生物学的"基因决定"原则）。**

微区列表：
- `tool_call` ✅ — 工具调用安全评估
- `payment` ✅ — 支付风险评估
- `shell_command` ✅ — Shell 命令安全（检测 rm -rf, sudo 等）
- `file_operation` ✅ — 文件操作安全（检测敏感路径、破坏性操作）

每个微区通过经验特化（符合"微区内突触可塑性"原则）。
所有微区共享同一个小脑引擎（RFF → K-head → Router），零核心代码改动。
33 个新测试，全部通过。MCP Server 已更新为 12 个工具。

### Phase 6：精密微操引擎 ✅ 已完成

**目标：让小脑能执行连续、实时、精细的操作——LLM 根本做不到的事。**

已实现的架构：

```
环境 (60Hz+)
    │ state
    ▼
┌─ MicroOpEngine ────────────────────────────────────────┐
│                                                          │
│  StateEncoder    →  数值状态 → 归一化向量 (<0.1ms)        │
│       │                                                  │
│  PatternSeparator →  RFF 稀疏编码 (复用核心引擎)          │
│       │                                                  │
│  ActionNet       →  状态编码 → 动作向量                   │
│       │                                                  │
│  ForwardModel    →  (state + action) → predicted_next    │
│       │                                                  │
│  ActionEncoder   →  解码动作向量 → 环境可执行动作          │
│       │                                                  │
│  SPE = actual - predicted  →  在线学习                    │
│                                                          │
└──────────────────────────────────────────────────────────┘
    │ action
    ▼
环境.execute(action) → reward
```

已实现组件：
1. **StateEncoder** (`core/state_encoder.py`) — 直接/投影两种模式，Welford 在线归一化
2. **ForwardModel** (`core/forward_model.py`) — 残差 MLP，预测状态变化量
3. **ActionEncoder** (`core/action_encoder.py`) — 编码/解码/字典接口
4. **MicroOpEngine** (`micro_ops/engine.py`) — 完整控制循环，集成以上所有组件
5. **TargetTracker** 环境 — 2D 移动目标追踪
6. **BalanceBeam** 环境 — 简化 CartPole 平衡控制
7. **34 个测试全部通过**

Demo 验证结果：
- 运行频率：**285 Hz**（远超 60Hz 目标）
- 单步延迟：**3.5ms**（远低于 16ms 预算）
- 前向模型 SPE：**降低 99%**（从 0.0187 → 0.0002）
- 零 LLM 调用 — 纯小脑计算
