## 一、我们到底在做什么

### 第一次叙事（已放弃）

~~"给 AI Agent 加个安全检查器"~~
- 这把小脑矮化成了一个二元分类器
- 与 AgentGuard、Fiddler、RADIUS 等竞品没有本质区别
- "更快更便宜"的价值主张随着 LLM 降价和 prompt cache 技术会自然贬值

### 第二次叙事（过渡阶段）

~~"给 LLM 一个身体——让它能积累经验、形成反射、自主成长"~~
- 方向正确，但"更快更便宜"仍然是隐含的主要价值主张
- Cursor 等编码 Agent 已经很流畅，用户找不到痛点
- 技能缓存的价值被 prompt cache 部分替代

### 当前叙事（第三次，也是最准确的一次）

> **当前 AI Agent 的核心问题不是太慢——是太脆弱。**
> **CMU 基准测试：最好的 Agent 成功率只有 24%。**
> **76% 的失败来自误差级联——一步错，步步错，而且 Agent 完全不知道自己错了。**
>
> **生物体不会这样。因为它有小脑。**
> **你端水走路时脚绊了，小脑在 50ms 内检测到姿态偏离预测、立即修正——**
> **皮层甚至不知道发生了什么。**
>
> **Digital Cerebellum 做同样的事：**
> **每一步预测后果，实时检测偏差，第一时间拦截错误，防止级联崩溃。**
> **不是让 Agent 更快——是让 Agent 不翻车。**

核心价值：

**把 AI Agent 的成功率从 24% 提升到 80%+，同时加速 100x。**

通过四个机制：
1. **预测性监控** — ForwardModel 每步预测后果，SPE 异常时立即暂停（100% 级联捕获）
2. **直觉预警** — SomaticMarker 从历史失败模式中提取"直觉"，在崩溃前发出警报
3. **技能自动化** — SkillStore 学过的模式直接执行，<10ms，不调 LLM（100x 加速）
4. **经验积累** — 每次交互都让小脑更快更准，跨会话持久化

## 二、为什么 Agent 这么脆弱（LLM 再强也解决不了的）

### 2026 年的残酷现实

| 数据来源 | 结论 |
|----------|------|
| CMU TheAgentCompany | Claude 3.5 Sonnet 完成率 **24%**，GPT-4o 仅 **8.6%** |
| 学术基准 | Agent 框架平均完成率约 **50%** |
| Gartner 预测 | **40%** 的 Agent 项目将在 2027 年前取消 |
| OpenClaw 用户反馈 | 单次操作延迟 **16+ 秒**，session 膨胀到 28000+ tokens |

### 根本原因：误差级联（Error Cascading）

```
Agent 执行 20 步任务：
  Step 1: ✅  →  Step 2: ✅  →  Step 3: ❌ (小错)
  Step 4: 基于错误状态继续 → 越来越偏
  Step 5-20: 全部失败
  Agent 全程没有意识到自己在第 3 步就错了
```

LLM 再强也解决不了这个问题，因为这是**架构层面的缺陷**：
- LLM 不记得上一步预测了什么结果 → 无法检测偏差
- LLM 没有"直觉"警报 → 无法在崩溃前预警
- LLM 不跨会话学习 → 同样的错反复犯

### 生物体为什么不会这样

| 场景 | 没有小脑 | 有小脑 |
|------|---------|--------|
| 端水走路，脚绊了 | 300ms 后皮层才反应，水洒了 | 50ms 内检测姿态偏差，立即修正 |
| 开车遇到路面结冰 | 等滑出去才知道 | 方向盘手感异常，预判并修正 |
| 弹钢琴弹错音 | 弹完才发现 | 手指触感不对的瞬间就修正 |

**小脑的核心功能不是"更快"——是"不翻车"。**

它通过前向模型持续预测：如果我这样做，结果应该是 X。
如果实际结果不是 X → 立即修正，不让错误传播下去。

### LLM 的本质局限

| 局限 | 说明 | 小脑如何解决 |
|------|------|-------------|
| **无预测监控** | 执行后不检查是否符合预期 | ForwardModel 每步预测 + SPE 检测 |
| **无直觉警报** | 异常模式无法触发"不对劲"的感觉 | SomaticMarker 从历史失败中学习直觉 |
| **无状态** | 每次对话从零开始 | 小脑是持久状态，跨会话积累经验 |
| **无身体** | 每个动作都要"想一步说一步" | 小脑学会动作序列，整套自动执行 |
| **误差不拦截** | 一步错，步步错 | SPE 异常 → 立即暂停 → 重新评估 |

## 三、五个核心能力

### 3.1 低延迟高效操作 ✅ 已实现

- 快路径 <10ms，benchmark 验证
- RFF 模式分离 + K-head 线性预测 = 数学上保证快

### 3.2 自主学习 ✅ 已实现（Phase 4）

- 在线学习（从 LLM 反馈中学习预测模型）
- 三通道误差信号（SPE/TPE/RPE）
- 记忆巩固（睡眠周期）
- EWC 防遗忘
- **SkillStore 程序性记忆** — 存储 (输入模式 → 输出动作/响应) 对
- **技能匹配** — 向量相似度检索，找到最匹配的已学技能
- **技能执行** — 高置信度时直接执行，不调 LLM
- **强化/消退** — 成功的技能增强，失败的技能减弱
- **睡眠巩固** — 合并相似技能，淘汰不可靠技能
- Demo 验证：20 次交互后 25% 自动化率

### 3.3 自由探索 ✅ 已实现（Phase 4）

- 好奇心驱动（追踪学习进展，识别高学习潜力领域）
- 探索排名（`get_curiosity_ranking()`）
- **主动探索信号** — `get_exploration_requests()` 输出可执行的探索建议
- **DigitalBrain.get_exploration_suggestions()** — 大脑级别的探索驱动

### 3.4 精密微操 ✅ 已实现（Phase 6）

**这是生物小脑最核心的能力。** 小脑的"本职工作"是精密运动控制——
认知功能（预测、安全检查）反而是近年才被广泛认识的"副业"。

**已实现：**
- **MicroOpEngine** — 连续控制循环：observe → predict → act → compare → learn
- **StateEncoder** — 数值状态向量直接进入 RFF（绕过 sentence-transformer，<0.1ms）
- **ForwardModel** — 前向模型 (state + action) → predicted_next_state（浅层前馈 MLP，匹配生物学）
- **ActionEncoder** — 连续动作空间编码/解码（归一化、pad/truncate）
- **PatternSeparator 复用** — 同一个 RFF 稀疏编码层，用于连续状态空间
- **在线学习** — 从感觉预测误差 (SPE) 实时更新前向模型
- **策略网络** — 基于奖励信号的在线策略梯度学习
- **两个验证环境** — TargetTracker（2D 追踪）、BalanceBeam（平衡杆）
- **Demo 验证**：285 Hz 实际运行速度，3.5ms/step，SPE 降低 99%

**应用场景：**

```
┌─────────────────────────────────────────────────────────┐
│  场景          大脑（LLM）说        小脑（微操）做        │
│  ─────────    ──────────────       ──────────────       │
│  MOBA         "团战集火ADC"        闪现+技能连招+走A      │
│  FPS          "守住B点"            压枪+预瞄+身法        │
│  Figma        "做一个登录页"        8px网格对齐+间距一致   │
│  代码          "重构这个模块"        精确的AST变换+重命名   │
│  机器人        "把杯子放桌上"        手臂轨迹+力度+平衡     │
│  钢琴          "弹月光奏鸣曲"        每个手指的力度和时机    │
└─────────────────────────────────────────────────────────┘
```

**技术需求：**
- 连续推理模式（streaming input/output，非 request-response）
- 动作空间编码（把连续动作映射为向量）
- 实时前向模型（预测动作的感觉后果）
- 序列学习（学习多步动作序列作为一个"技能单元"）
- 时序精度（毫秒级动作协调）

**生物学映射：**

| 生物学 | 数字系统 |
|--------|---------|
| 运动皮层发出运动指令 | LLM 发出高层意图 |
| 小脑接收传出副本 | 小脑接收 LLM 的意图 |
| 小脑通过前向模型预测感觉后果 | 小脑预测动作结果 |
| 小脑输出精细运动修正 | 小脑输出具体操作序列 |
| 攀爬纤维传递误差 | 执行结果反馈驱动学习 |
| 反复练习 → 程序性记忆 | 在线学习 → 自动化执行 |

**这是我们与"小 LLM 做 System 1"方案最根本的区别：**
小 LLM 能做"快速判断"，但做不了"精密控制"。
只有小脑架构——浅层前馈、实时闭环、序列学习——才能做到微操。

### 3.5 预测性错误拦截 ✅ 已实现（Phase 7）

**这是小脑对 Agent 可靠性最大的贡献。**

生物学基础：
- 小脑持续运行前向模型，预测每个动作的感觉后果
- 当实际结果偏离预测（SPE 升高）→ 攀爬纤维 burst → 立即修正
- 这就是为什么你端水走路脚绊了不会摔倒——小脑在 50ms 内拦截了错误

数字实现（已完成 — Phase 7a）：
- **StepMonitor** ✅ — 包裹 Agent 的每一步操作，执行前预测、执行后比较
- **ErrorCascadeDetector** ✅ — 跟踪连续 SPE，检测误差是否在累积
- **StepForwardModel** ✅ — 文本/向量通用前向模型，预测步骤结果
- **FailureMemory** ✅ — 记录失败模式，下次遇到直接预警
- **Save/Load** ✅ — 持久化前向模型 + 失败记忆，跨会话积累经验
- **Reliability Benchmark** ✅ — 7 场景对比，100% 级联捕获，71% 浪费预防
- **AutoRollback** ✅ — 级联检测后自动计算回退方案（RollbackPlan：回退到哪步、哪些步骤浪费了、恢复建议）
- **LangChain 集成** ✅ — CerebellumCallback 一行接入，级联时抛 CerebellumPause 异常
- **MCP 工具扩展** ✅ — 17 个工具（含 monitor_rollback_plan）

应用场景：
```
桌面 Agent 执行 "把文件从 A 移到 B"：

  Step 1: 打开文件管理器 → ForwardModel 预测窗口出现 → ✅ 匹配
  Step 2: 导航到 A 文件夹 → ForwardModel 预测看到文件列表 → ✅ 匹配
  Step 3: 选择文件 → ForwardModel 预测文件高亮 → ⚠️ SPE 异常！
          文件没有高亮 → 可能路径错了 / 文件不存在
          → 立即暂停，不执行 Step 4 (剪切)
          → 截图发给 LLM 重新评估
          → 避免了一次 "剪切了错误文件" 的灾难
```

### 已纠正的错误

**错误：小脑微区可以通过经验自主涌现**

> 事实：小脑微区是基因决定的，不是经验涌现的。
> "The molecular identity of individual Purkinje cells seems to be
> set and independent of cerebellar connectivity."
> — Nature Reviews Neuroscience

- 微区结构在出生前由基因编程，在个体间高度一致、跨物种保守
- 经验只改变微区**内部**的突触权重，不改变微区的**存在与否**
- 正确做法：预定义通用微区 → 通过经验特化（而非从零涌现）

### 符合生物学的技能习得模型

来自 2025 Nature Communications 的运动学习三阶段：

```
阶段 1（早期）: 小脑主导
  → 快速检测误差，调整动作
  → 对应：慢路径，LLM 处理，小脑学习

阶段 2（中期）: 皮层-小脑协同
  → 双向交互，逐步内化
  → 对应：shadow 路径，小脑预测 + LLM 验证

阶段 3（晚期）: 技能自动化
  → 小脑抑制减少，皮层自主执行
  → 对应：快路径接管，不需要 LLM

记忆巩固: 从小脑皮层 → 小脑核团
  → 从复杂灵活系统 → 简单快速系统
  → 对应：睡眠周期将技能压缩为程序性记忆
```

## 七、核心洞察：所有人都在用皮层模拟小脑

### 行业现状

2026 年，整个 AI Agent 生态都在尝试解决同样的问题——路由、缓存、自我纠正、安全检查。
但**每一个方案都是用 LLM（皮层）来模拟小脑的功能**：

| 行业方案 | 模拟的小脑功能 | 代价 | 我们怎么做 |
|---------|--------------|------|----------|
| **RouteLLM** — 训练一个模型来选择走哪个 LLM | 置信度路由（深部核团 DCN） | 需要离线训练数据，不在线学习 | Population Coding + 在线 RPE 调整阈值 |
| **GPTCache** — 缓存过去的 LLM 响应 | 程序性记忆（SkillStore） | 精确匹配才命中，不泛化 | SkillStore：相似模式即可泛化 |
| **CoT 自我纠正** — 让 LLM 检查自己的输出 | 前向模型（预测→比较→修正） | 完整 LLM 推理，1-3秒/次 | 浅层 MLP，<5ms |
| **Guardrails / LlamaFirewall** — 静态规则或 LLM 评估 | 安全微区 | 规则不学习，LLM 调用昂贵 | 在线学习微区，越用越准 |
| **MemGPT** — 用 LLM 管理自己的记忆 | 状态维护 | 消耗 context window token | 持久化存储，零 token 开销 |
| **Self-reflection Agent** — 让 Agent 反思自己的表现 | 元认知（SelfModel） | 每次反思 = 一次完整推理 | 逐域 ECE 校准，实时计算 |

### 生物学类比

这就像**小脑损伤的病人用皮层代偿**：

```
正常人端水走路：
  小脑: 持续预测姿态 → 检测偏差 → 50ms 修正
  皮层: 安心想事情、聊天
  → 水稳稳的，同时还能说话

小脑损伤的病人端水走路：
  皮层: 必须全部注意力盯着水杯 → 有意识地调整每一步
  → 慢（300ms 才反应）
  → 累（消耗全部工作记忆）
  → 脆（有人跟你说话就会洒）
  → 不学习（练多少次都不会自动化）
```

**当前 AI Agent 生态就是那个"小脑损伤的病人"。**

每一个"优化"都是用皮层来代偿小脑功能：
- **慢** — 每次自我纠正/路由/检查 = 一次完整 LLM 推理
- **贵** — 占用 context window，消耗 token
- **不学习** — 下次遇到同样的问题还是从头来
- **脆弱** — 认知负载（复杂任务）一高就崩

### 我们的区别

**我们不是在用皮层模拟小脑——我们直接实现了小脑。**

| 维度 | 皮层模拟小脑（行业现状） | 真正的小脑（我们） |
|------|----------------------|------------------|
| **延迟** | O(LLM 推理) ≈ 1-10秒 | O(浅层前馈) ≈ 5ms |
| **学习** | 不学习 / 需要离线训练 | 在线学习，每次交互都改进 |
| **记忆** | 在 context window 内 | 持久化，跨会话 |
| **成本** | 每次检查 = token 消耗 | 一次本地计算，成本趋近于零 |
| **并行** | 占用 LLM 推理能力 | 独立运行，LLM 可以同时思考 |
| **泛化** | 缓存不泛化 / 规则不适应 | 从经验中泛化到新模式 |
| **运动控制** | 无法实现 | 285Hz 连续控制 |

### 为什么这不只是"更好的优化"

这是**架构范式转换**。

生物进化没有选择"让皮层更大来替代小脑"——它进化出了小脑作为专用子系统。
因为有些计算——预测、时序、纠错、运动自动化——天然适合用不同于思辨推理的架构来做。

我们的工作证明，同样的原理适用于数字 Agent：
**不是更大更快的 LLM，而是一个专用的、轻量级的、在线学习的小脑层。**
